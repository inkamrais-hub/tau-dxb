# 表征调制驱动的自监督 RL 框架 — 初步报告

> 2026-07-02 · 状态：概念阶段，待 erp3 训练完成后验证

## 一句话

erp3（τopus + WTA 竞争稀疏 + 剪枝）已经让模型"每层每头自己调听多准"。
本框架想再加一层：用层间表征当传感器，RL 读取表征异常，推模型把调得不好的地方改对。
目标不是刷 CER，是**保障泛化**——为复赛藤县方言迁移打底。

## 已有资产（不用从零造）

| 组件 | 位置 | 作用 |
|------|------|------|
| τopus | `patch_whisper_stau.py` | 每 head 可学习 τ/α，控制注意力聚焦度 |
| WTA 竞争稀疏 | `patch_whisper_sparse_grad.py` | FFN 中间层 top-k 激活，前向稀疏 |
| 动态剪枝 | 同上 | win rate < 5% 的神经元冻结，允许复活 |
| DualProbe | `f:\τ\repeng\repeng_toolkit\drm.py` | D(l) = cos(Δh_in, Δh_out)，检测信息流断裂层 |
| FlowDecomposer | 同上 | 把 hidden state 分解为输入流/输出流/内部流 |
| HiddenStateGuider | `steering.py` | h + α(h_correct − h_wrong)，CFG 式表征引导 |
| ActivationSteerer | 同上 | 向指定层注入方向向量 |
| XXT 自然控制轴 | `_qwen_xxt_representation_steering.py` | PCA 提取残差流自然轴 |

**关键经验发现**：EXP-DR-1 在 Qwen2.5-0.5B 上观测到 D(l) 在某层突降，同时 χ 跳变 + eff_dim 相变，三者同步。这是"层间表征断裂可被检测"的 [FACT] 级证据（但仅限文本 LLM，未在 ASR 验证）。

## 框架雏形

```
┌─────────────────────────────────────────────────┐
│  erp3（执行器）                                    │
│  τopus 每头调聚焦 · WTA 选赢家 · 剪枝淘汰         │
│  → 模型已能逐层逐头自我调节                        │
├─────────────────────────────────────────────────┤
│  层间表征传感器（新增）                            │
│  读取 encoder/decoder 各层表征                    │
│  检测：哪里断裂？哪里泛化间隙大？                   │
│  → 输出一个"调节得好不好"的信号                    │
├─────────────────────────────────────────────────┤
│  RL 回路（新增）                                   │
│  传感器信号 → reward                              │
│  推 erp3 把调得不好的地方改对                      │
│  约束：改了不能破坏已有的低 CER                    │
└─────────────────────────────────────────────────┘
```

### 传感器的候选定义

| 方案 | 信号来源 | 优点 | 风险 |
|------|---------|------|------|
| D(l) 对偶度 | 音频扰动 vs 文本扰动，各层表征变化 | 你们已验证能检测断裂 | ASR 上对偶定义需适配 |
| 层间分歧度 | 相邻层表征的余弦距离 | 最简单，零额外前向 | 断裂 ≠ 错误，需验证相关 |
| 训练/dev 表征分布偏移 | 两集 D 分布的 KL | 直接瞄准泛化间隙 | 需要两个数据集的前向 |

### "对抗"的含义

不是两层互相 GAN 式博弈。是：
- **训练集的层间协调性 vs dev 集的层间断裂** 这对对立
- 模型在训练样本上"层间协调"，到新样本就"断裂" → 泛化不好
- RL 推模型去缩小这个间隙

## 命门假设

> **层间表征断裂是过拟合/泛化差的几何征兆，而非中性结构。**

- 如果成立 → 拿断裂当约束能促进泛化，框架有理论基础
- 如果不成立 → 断裂只是噪声，拿它当约束是优化无关目标

**这个假设目前只有观测性相关（EXP-DR-1），没有因果验证。**

## 命门验证实验（erp3 训练完后跑）

**实验 LK-001：层间断裂 ↔ 泛化/CER 相关性**

1. 拿 erp3 最终权重，对训练集 + dev 集前向，抽每层每个时间步 hidden state
2. 算层间分歧度 D(t, layer)（最简单用相邻层余弦距离）
3. 看两件事：
   - **相关**：D 高的时间步 → CER 是否也高？AUC > 0.7 才说明断裂跟错误有关
   - **泛化间隙**：训练集 D 分布 vs dev 集 D 分布是否有显著偏移

**判据**：
- 两个都成立 → 假设升级为 [FACT]，继续设计 RL
- 只有相关、无泛化间隙 → 降级为"错误检测器"而非"泛化保障器"
- 都不成立 → 方向 falsified，止损

**开销**：几小时，零额外训练，纯前向 + 统计。

## "炸鱼"评估

| 视角 | 判断 | 理由 |
|------|------|------|
| 纯工程降 CER | 炸鱼 | 数据增强/LM融合/后处理更直接，RL+表征复杂度高收益不确定 |
| 决赛创新性(10%) | 不炸鱼 | 完整故事：τopus+WTA+剪枝+表征RL，技术深度高 |
| 复赛泛化保障 | 值得赌 | 藤县方言 20GB 从 0 训练，泛化是生死线 |
| 独立研究价值 | 不炸鱼 | 表征驱动 RL 是有发表潜力的方向 |
| 时间风险 | 高 | erp3 还在跑，复赛 8/4 开始，验证周期可能一周+ |

**定位建议**：增量实验，不阻塞主线。erp3 训练完 → 跑 LK-001 → 过了再继续，不过就撤。
